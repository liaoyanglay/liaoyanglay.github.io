ConstraintLayout解析

发表于 2018-11-11 | 分类于 Android

介绍

什么是ConstraintLayout？官方文档是这样说的：

A ConstraintLayout is a ViewGroup which allows you to position and size widgets in a flexible way.

看起来好像和RelativeLayout很像，那我们为什么要使用它呢？其实，ConstraintLayout比起RelativeLayout还是要强大很多的。主要体现在两个方面，一个是更加灵活，还有一个就是当布局比较复杂时，它的性能会好很多。

我们都知道，View的绘制主要会经历三个阶段，measure，layout和draw，这三个过程都需要自顶向下遍历视图树，因此布局结构嵌套的越深，系统绘制view的开销也就越大。有时候一个复杂的布局用传统的写法需要嵌套到七，八层甚至更多，而且RelativeLayout会对每个子View重复测量两次，如此庞大的开销甚至会导致丢帧的现象。此时ConstraintLayout的优势就体现出来了，它可以非常容易的创建一个具有扁平结构的复杂布局，无需任何嵌套。性能上的优势足以说服我们使用ConstraintLayout。

用法

ConstraintLayout在使用时既可以用代码来编写，也可以用可视化布局来拖拽。我们先简单说说后一种。和以往不太一样，ConstraintLayout用可视化方式编写界面其实还是挺友好的，只需要简单的拖拽即可添加约束。并且可以在工具栏打开自动添加约束的功能，不过有时候会和想要的效果不太一样，需要自己调整。对于习惯了代码编写界面的人，一开始用可视化拖拽会很不适应，熟悉以后其实效率还不错。不过个人还是更偏爱用代码编写。

先介绍一下常用的属性。

Constraint

首先是约束，这也是ConstraintLayout名字的由来，view的四个方向和文本基线都可以添加约束，它的作用就是标识相对位置。只有相同的方向之间才能添加约束，水平方向和竖直方向之间是无法添加约束的。

当控件具有固定的宽高或者将其设为WRAP_CONTENT时，约束一般不会改变控件的大小。那如果想要用约束决定空间的大小该怎么办呢？可以将宽高设置为MATCH_CONSTRAINT，即0dp，这时控件的大小就会跟着约束改变了。并且官方建议使用MATCH_CONSTRAINT来代替以前的MATCH_PARENT，可以达到相同的效果。

Margins

控件也和以前一样可以在各个方向添加Margin，不过ConstraintLayout又添加了几个新的属性，叫做layout_goneMargin，它有什么用呢？还是以官方的图来说明

假如我们将B控件的左边约束到A的右边，当A的visibility属性设置为gone时，ConstraintLayout做了一些特殊的处理，此时B仍然受到A的约束。由于A不会在布局中占用任何空间，所以可以将A视为一个所有margin都等于0的点，B受到约束的效果如图。此时B的margin仍是连接到A时的大小，如果B设置了layout_goneMargin属性，则margin会变为这个属性的大小。

将控件标记为gone的行为不会破坏布局，而且在进行简单的布局动画时非常有用。

bias,ratio和percent

我们可以把约束比作一根橡皮筋，默认情况下所有橡皮筋的拉力都是相等的。当两边都添加约束后，控件就会处于约束中心的位置。通过设置bias属性就可以改变两侧“橡皮筋”的弹力，从而改变控件的位置。

如果想指定控件的宽高比，就需要用到ratio属性。它要求至少将一个尺寸设置为0dp（即MATCH_CONSTRAINT）。如果宽和高都设置为0dp，空间的尺寸会是满足所有约束的最大尺寸，且保持宽高比。还可以在宽高比之前加上“W,”或”H,”来指定哪个方向被比率约束。如：

当控件的尺寸设置为MATCH_CONSTRAINT时（0dp），可以通过两个属性来设置该控件占父控件的百分比，值为0-1。

Circular positioning

圆形定位，这是在1.1版本中新添加的简单实用功能。可以让一个控件以另一个控件的中心为圆心，通过设置相对的距离和角度实现定位。这个属性配合布局动画也让一些原本复杂的效果变得很容易实现。

Chains

如果一系列控件双向的连接起来，就构成了一条链，这是一种特殊的约束。我们可以通过链来实现比LinearLayout更复杂的效果。

链有水平和竖直两个方向，最左边或最上边的控件是链头，可以设置链的style属性。下面介绍一下几种style，分别是spread，spread_inside和packed。

默认是spread，即所有的控件都会分散开来。在这种模式下如果将控件的尺寸设置为0dp，整个链将会占满所有可用空间，通过设置weight属性，可以实现和LinearLayout相同的效果

spread_inside，类似于spread，只不过链的两边不会分散。

packed,顾名思义，把链中所有控件打包，可以把他们看作一个整体，此时可以通过链头设置bias属性来约束所有控件的位置。

到这里一些基本的用法就讲完了，其实ConstraintLayout用熟练以后还是很容易做出想要的效果，至少用它来替代RelativeLayout和LinearLayout是没什么问题的，然而它的强大之处还不止于此。

动画

ConstraintLayout扁平化布局所带来的另一个好处就是控件的对齐容易了许多。有些动画如果要在很深的嵌套布局里实现，可能会遇到各种各样的问题。而扁平化的设计，很适合各种线性动画的实现。

先介绍一下和ConstraintLayout紧密相关的ConstraintSet这个类，由官方文档：

This class allows you to define programmatically a set of constraints to be used with ConstraintLayout. It lets you create and save constraints, and apply them to an existing ConstraintLayout.

ConstraintSet保存着一系列约束的集合，它允许添加和清除约束，并可以把这些约束应用到一个已经存在的ConstraintLayout。

其他

到这里ConstraintLayout的介绍差不多结束了，其实它的功能还不止这些，比如Guideline，Barrier，Placeholder，Optimizer等，这里就不详细介绍了，感兴趣的小伙伴可以自己去试试。ConstraintLayout自从去年Google I/O大会推出以来，功能一直在完善，也更加好用。应该会在界面编写中占据越来越重要的地位，期待能用它来做出一些更有意思的东西。
